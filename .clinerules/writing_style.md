# Coding Principles

## Modular Design
Always write highly modular code. Break down large functionalities into small, single-purpose, and reusable components or functions. Avoid creating massive, monolithic blocks of code or "big blob" files that are difficult to read and maintain. Keep every file focused on a single responsibility.

## Minimalist Commenting
Code should be self-explanatory through clear naming conventions and intuitive structure. Keep comments to an absolute minimum. Only use them when explaining the "why" behind complex business logic or non-obvious workarounds, not the "what" or "how" of standard implementation.

## Best Practices
Adhere strictly to industry standards and language-specific best practices. Write clean, idiomatic, and modern code, ensuring proper error handling, type safety, and optimal performance throughout the project. 

## Best Practices 2
Adhere strictly to industry standards and modern programming practices. Follow Clean Code principles inspired by Robert C. Martin (Uncle Bob): ensure code is highly readable, functions do only one thing, and logic is self-documenting. Write clean, idiomatic code with proper error handling, type safety, and optimal performance.
